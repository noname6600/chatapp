## ADDED Requirements

### Requirement: Server-managed createdAt for message-domain entities
The system SHALL generate `createdAt` on the server at persistence time for `ChatMessage`, `ChatAttachment`, `ChatMessageMention`, and `ChatReaction`. Request payloads MUST NOT be required to provide timestamp fields.

#### Scenario: Message request has no timestamp and persists successfully
- **WHEN** a message send request is submitted without any time field
- **THEN** the persisted `ChatMessage.createdAt` is non-null and generated by the server

#### Scenario: Attachment createdAt is generated on save
- **WHEN** attachments are persisted as part of message send flow
- **THEN** each persisted `ChatAttachment.createdAt` is non-null and server-generated

#### Scenario: Mention createdAt is generated on save
- **WHEN** mentions are persisted in mention persistence step
- **THEN** each persisted `ChatMessageMention.createdAt` is non-null and server-generated

#### Scenario: Reaction createdAt is generated without pipeline assignment
- **WHEN** a new reaction is persisted
- **THEN** persisted `ChatReaction.createdAt` is non-null even if pipeline code does not set it explicitly

### Requirement: Timestamp generation strategy is consistent across entities
The system SHALL use one timestamp generation strategy for all four entities in this module, and application/pipeline code SHALL NOT manually assign `createdAt` for only a subset.

#### Scenario: Reaction persistence path matches message entity strategy
- **WHEN** reaction persistence executes
- **THEN** it follows the same server-side lifecycle timestamp model used by message/attachment/mention entities

### Requirement: Events publish persisted timestamps
The system SHALL publish timestamps from persisted entities, not from request/context placeholders, for message and reaction events.

#### Scenario: Message event payload includes persisted createdAt
- **WHEN** message-created event is published after persistence
- **THEN** event payload `createdAt` equals the persisted message `createdAt` and is non-null

#### Scenario: Reaction event payload includes persisted createdAt
- **WHEN** reaction event is published for a newly added reaction
- **THEN** payload `createdAt` is non-null and sourced from persisted reaction state
